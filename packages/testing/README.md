# packages/testing

Fixtures, mocks y test helpers.

Responsabilidad declarada:
- alojar fixtures declarados;
- diferenciar datos simulados de datos observados;
- ayudar a validar contratos sin fuentes reales.

Limites:
- no usa datos productivos;
- no conecta servicios externos;
- no escribe DB;
- todo fixture debe declarar su fuente.
